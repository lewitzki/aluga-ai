# 11 - Cliente painel de emprestimos e historico

## Objetivo
Construir painel do cliente com emprestimos ativos, historico e total pago.

## Escopo tecnico
- Criar pagina Inertia para painel cliente.
- Listar emprestimos por status (`scheduled`, `active`, `finished`, `late`).
- Exibir resumo financeiro acumulado.

## Criterios de aceitacao
- [ ] Cliente visualiza seus emprestimos ativos com prazo correndo.
- [ ] Cliente visualiza historico de ferramentas emprestadas.
- [ ] Cliente visualiza total pago com base em pagamentos aprovados.
- [ ] Dados respeitam escopo do usuario logado.

## Dependencias
- `10-validacao-de-disponibilidade-por-periodo.md`
